### Mục tiêu
GPU OOM prevention + E2E GPU stability & system resilience tests — đảm bảo worker không bị OOM khi khởi tạo model và xử lý liên tục.

### Files thay đổi

**New files:**
- `03_worker/app/utils/gpu_memory.py` — GPU memory management utilities:
  - `set_gpu_memory_fraction()`: Giới hạn GPU memory fraction trước khi load model (default 85%, configurable via `GPU_MEMORY_FRACTION` env)
  - `check_gpu_available(min_free_mb)`: Kiểm tra GPU memory trước khi init model
  - `log_gpu_memory(context)`: Log GPU allocated/reserved/free/total
  - `cleanup_gpu_memory()`: Aggressive cleanup — Python GC + `paddle.device.cuda.empty_cache()`
  - `get_gpu_memory_info()`: Trả về dict chi tiết GPU memory status
  - PaddlePaddle flags: `FLAGS_fraction_of_gpu_memory_to_use`, `FLAGS_initial_gpu_memory_in_mb=256`, `FLAGS_eager_delete_tensor_gb=0.0`

- `00_test/e2e/test_e2e_gpu_stability.py` — 15 GPU stability tests (GS-001→GS-015):

  - **Heavy Load** (3 tests): multi-page PDF stress, batch 3 files, structured PDF (heaviest GPU load)
  - **Cross-Engine** (3 tests): text→VL, VL→text, 5 rounds interleaved
  - **Post-Load** (1 test): health + smoke test after all previous tests

- `00_test/e2e/test_e2e_system_resilience.py` — 16 resilience tests (SR-001→SR-016):
  - **Error Recovery** (4 tests): invalid MIME, corrupt image, recovery after error, empty PDF

  - **Service Stability** (3 tests): health consistency, service list consistency, request list after load
  - **Output Formats** (2 tests): all text formats, all structured formats

**Modified files:**
- `03_worker/app/engines/paddle_text/handler.py`:
  - Import `gpu_memory` utils
  - `__init__`: `set_gpu_memory_fraction()` + `check_gpu_available(400MB)` + `log_gpu_memory()` before/after init
  - `process()`: Dùng `cleanup_gpu_memory()` thay vì raw `paddle.device.cuda.empty_cache()`, thêm log sau processing

- `03_worker/app/engines/paddle_vl/handler.py`:
  - Import `gpu_memory` utils
  - `__init__`: `set_gpu_memory_fraction()` + `check_gpu_available(800MB)` (PPStructure cần nhiều hơn) + `log_gpu_memory()` before/after init
  - `process()`: Dùng `cleanup_gpu_memory()`, thêm log, final cleanup sau khi xử lý xong

- `03_worker/app/core/worker.py`:
  - Import `cleanup_gpu_memory`, `log_gpu_memory`
  - `process_job()` finally block: cleanup GPU memory + log sau mỗi job

### Lý do
**Vấn đề:** Worker GPU (đặc biệt PPStructureV3) load nhiều model cùng lúc (layout + table + OCR) có thể gây OOM. Xử lý PDF multi-page tích luỹ GPU memory. Không có monitoring hay giới hạn memory fraction.

**Giải pháp 3 lớp:**
1. **Prevention**: `FLAGS_fraction_of_gpu_memory_to_use=0.85` + `FLAGS_initial_gpu_memory_in_mb=256` giới hạn PaddlePaddle không chiếm hết GPU. Kiểm tra memory trước khi init model.
2. **Cleanup**: `cleanup_gpu_memory()` gọi GC + `empty_cache()` sau mỗi page và mỗi job. PPStructure VL handler có thêm final cleanup.
3. **Monitoring**: Log GPU memory tại 4 điểm: trước/sau init model, sau processing, sau mỗi job. Dễ dàng phát hiện memory leak qua logs.

**E2E tests** verify toàn bộ: worker startup OK (model loaded), sequential processing không leak, heavy PDF processing, cross-engine stability, error recovery không gây GPU leak.

Tổng E2E tests: 118 → **149 tests** (+15 GPU stability + 16 resilience).

## time: 14:58

### Mục tiêu
Phân tích tài liệu `04_docs/05-OCR_capabilities/text_extract_improvement` (OCR Workers Technical Specification) và đối chiếu với codebase worker hiện tại (`03_worker/`) → xây dựng kế hoạch triển khai cải tiến OCR text extraction.

### Phân tích Gap: Đề xuất vs. Hiện trạng

| Đề xuất trong spec | Worker hiện tại | Mức Gap |
|---|---|---|
| Quality Check module (8 checks: blur, brightness, noise, skew, orientation, perspective, resolution, border) | Chỉ `paddle_vl` check kết quả OCR garbage → fallback Tier 2. Không check chất lượng ảnh đầu vào | **Lớn** |
| Preprocessing module (8 bước: orientation fix, perspective, deskew, border removal, upscale, denoise, CLAHE, binarization) | Chỉ resize/convert RGB cơ bản | **Lớn** |
| `ocr_text_raw (PP-OCRv5)` — text thuần, lang=vi, batch=16, VRAM 0.5GB | `paddle_text` — PaddleOCR v3.x, đã có GPU memory mgmt | Nhỏ |
| `ocr_text_formatted (Marker/Surya)` — layout-preserved, MD/HTML output, VRAM ~5GB | `paddle_vl` — PP-Structure, layout analysis khác engine | **Lớn** |
| `ocr_text_styled (Tesseract + OpenCV)` — 4 module pipeline: hOCR + style analysis (7 thuộc tính) + layout reconstruction + DOCX/HTML generation | `tesseract` — chỉ extract text + bbox, output txt/json | **Rất lớn** |

### Kế hoạch triển khai (5 items, ưu tiên theo impact/effort)

#### Priority 1: Quality Check Module (shared) — ~3-4 ngày

**Lý do ưu tiên cao nhất**: Cải thiện accuracy cho TẤT CẢ workers, chạy CPU (0 VRAM), low risk.

**Scope**: 8 kiểm tra chất lượng ảnh đầu vào
- Blur detection (Laplacian variance, threshold ~80-100)
- Brightness & contrast (mean/std trên grayscale)
- Noise estimation (MAD trên Laplacian)
- Skew detection (Hough Lines + Projection Profile consensus)
- Orientation detection (Tesseract OSD `--psm 0` + heuristic backup)
- Perspective distortion (contour tứ giác > 30% diện tích)
- Resolution check (DPI estimation giả định A4)
- Black border detection (mean pixel 4 dải cạnh)

**Scoring**: Điểm khởi đầu 1.0, trừ theo trọng số (blur -0.50, contrast -0.20, brightness -0.15, DPI -0.15, noise -0.10, perspective -0.10, skew -0.05, orientation -0.05, border -0.02).

**Output**: `quality_score` (0-1), `level` (good/acceptable/poor/reject), `issues[]`, `recommendation`.

**Vị trí code**: `03_worker/app/engines/shared/quality_check.py`

**Thay đổi backend cần thiết**: Mở rộng job status update để trả quality info → frontend hiển thị warning → user CANCEL/IGNORE.

#### Priority 2: Preprocessing Module (shared) — ~3-4 ngày

**Phụ thuộc**: Quality Check (chỉ sửa những gì đã flag).

**Scope**: 8 bước, thứ tự geometry trước → pixel sau
1. Orientation fix (xoay 90°/180°/270°)
2. Perspective correction (4-point warp)
3. Deskew (affine rotation, BORDER_REPLICATE)
4. Border removal (threshold + crop + padding 5px)
5. Resolution upscale (bicubic, cap 4x, target 300 DPI)
6. Denoise (bilateral filter d=9, sigmaColor=75, sigmaSpace=75)
7. Contrast enhancement (CLAHE trên L channel LAB, clip 8×8)
8. Binarization (CHỈ cho Tesseract worker — adaptive Gaussian threshold 11×11, C=2)

**Resize cho VRAM**: Max 2048px cạnh dài (`INTER_AREA`) — thay thế logic resize riêng lẻ hiện tại của mỗi engine.

**Vị trí code**: `03_worker/app/engines/shared/preprocessing.py`

**Lưu ý**: Phải tương thích với resize logic hiện tại của `paddle_vl` (upscale <960px, downscale >1280px). Preprocessing mới sẽ thay thế logic cũ.

#### Priority 3: Upgrade PP-OCRv5 — ~1 ngày

**Quick win**: PP-OCRv5 cải thiện accuracy +13% so với v4.

**Scope**:
- Update PaddleOCR package version
- Config: lang=`vi`, recognition batch=16, GPU memory limit 2GB
- Set `FLAGS_allocator_strategy=auto_growth`
- Verify API compatibility (handler hiện tại đã support v3 `predict()` + v2 `ocr()`)

**VRAM**: ~0.5 GB — rất thoải mái trên 8GB card.

**Rủi ro**: API có thể thay đổi nhẹ → cần test.

#### Priority 4: Thêm Marker Engine (formatted output) — ~5-7 ngày

**Scope**: Engine mới dùng Marker (Surya OCR) cho layout-preserved output.

**Pipeline**: detect → layout → recognize → table → assemble Markdown.

**Output formats**: `md`, `html`, `json`.

**VRAM tuning cho RTX 3070**:
- Batch size: 8 (detection, recognition, layout) — set env TRƯỚC import
- `DETECTOR_BATCH_SIZE=8`, `RECOGNITION_BATCH_SIZE=8`, `LAYOUT_BATCH_SIZE=8`
- `INFERENCE_RAM=8`, `TORCH_DEVICE=cuda`
- Peak ~5GB — tight nhưng an toàn trên 8GB card

**Vị trí code**: `03_worker/app/engines/marker/` (handler, preprocessing, postprocessing)

**Deploy**: `03_worker/deploy/marker/` (Dockerfile, docker-compose, requirements)

**Chiến lược**: Thêm song song với `paddle_vl`, KHÔNG thay thế. Benchmark Vietnamese test set rồi quyết định engine nào làm default.

**Rủi ro**: Marker chưa verify Vietnamese kỹ; VRAM tight (5GB/8GB); env vars phải set ở Dockerfile level.

#### Priority 5: Upgrade Tesseract → Styled Pipeline — ~7-10 ngày

**Scope lớn nhất**: Xây 4 module pipeline hoàn toàn mới, toàn bộ CPU.

**Module 1 — Tesseract hOCR**: Switch sang hOCR mode → word-level bbox + confidence + font size hint. Config: `vie`, `--dpi 300`, psm 3.

**Module 2 — OpenCV Style Analysis** (7 thuộc tính):

| Thuộc tính | Phương pháp | Accuracy |
|---|---|---|
| Font size | `bbox_height_px / DPI × 72` | ~90% |
| Bold | Distance Transform → stroke width > 2.5px@300DPI | ~85% |
| Italic | Line fit → góc > 8° so với phương đứng | ~80% |
| Underline | Canny + Hough ở 15% dưới bbox | ~75% |
| Color | Median RGB qua binary mask | ~95% |
| Super/subscript | So sánh y-position với baseline + font size nhỏ hơn | ~70% |
| Font family | Heuristic serif/sans-serif/monospace (3 nhóm) | ~50% |

**Module 3 — Layout Reconstruction**: Column detection (x-projection histogram) → reading order sort → paragraph grouping (>1.5× line_height) → header/footer detection.

**Module 4 — Document Generation**: python-docx cho DOCX (runs with styled text), HTML với inline CSS.

**Đề xuất implement theo phase**:
- Phase A (ưu tiên): hOCR + font size + bold + color + DOCX generation → cover 80% use case
- Phase B (nice-to-have): italic + underline + super/subscript + font family

**Rủi ro Vietnamese**: Italic false positive do dấu ngã; font size sai do stacked diacritics tăng bbox height; font name chỉ 50% accuracy.
