## 1. HIỆN TRẠNG HỆ THỐNG

### Kiến trúc tổng quan
Hệ thống OCR local MVP 1.0.0 đã được xây dựng hoàn chỉnh với kiến trúc **3 lớp**:

| Lớp | Thành phần | Trạng thái |
|-----|-----------|------------|
| **Edge** | React 18 Frontend, FastAPI Backend (port 8000), MinIO Storage | Hoạt động |
| **Orchestration** | Auth, Upload, Job, File Proxy, Admin module, NATS JetStream, SQLite | Hoạt động |
| **Processing** | PaddleOCR (GPU), Tesseract (CPU), PaddleOCR-VL (GPU) | Hoạt động |

### Các tính năng đã hoàn thành
- [x] Đăng ký / đăng nhập (email + password, JWT)
- [x] Phân quyền Admin / User
- [x] Upload file theo batch (validation mime type, dung lượng)
- [x] Xử lý OCR qua 3 engine: PaddleOCR, Tesseract, PaddleOCR-VL
- [x] Worker tự động đăng ký với backend khi khởi động (self-registration)
- [x] Worker giao tiếp file thông qua File Proxy (không truy cập storage trực tiếp)
- [x] Hàng đợi xử lý với NATS JetStream (phân luồng theo method/tier)
- [x] Admin panel: dashboard, quản lý services (approve/reject/disable), quản lý user
- [x] Tách deploy worker riêng biệt (docker-compose riêng cho từng engine)
- [x] Tái cấu trúc thư mục: 01_frontend, 02_backend, 03_worker, 04_docs
- [x] Frontend hiển thị đúng service đã được admin approve cho user

### Hạ tầng
- **Container:** Docker Compose (infra + backend chung, workers tách riêng)
- **Message Queue:** NATS JetStream (ocr-nats:4222)
- **Object Storage:** MinIO (ocr-minio:9000)
- **Database:** SQLite (ocr_fresh.db) qua SQLAlchemy 2.0
- **Network:** Docker bridge network `ocr-network`
