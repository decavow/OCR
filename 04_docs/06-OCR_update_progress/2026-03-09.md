## Observability Audit & Implementation Plan

### Bối cảnh

Paddle-vl worker đăng ký với `allowed_methods=["ocr_text_raw"]` thay vì `["structured_extract"]` do env override lỗi. Không ai phát hiện vì:
- Registration endpoint không có log → không thấy methods sai
- Admin approve mà không biết methods đang lưu là gì → không có audit trail
- Heartbeat, job_status, file_proxy endpoints cũng hoàn toàn im lặng

→ Cần triển khai observability để phát hiện và debug các vấn đề tương tự.

#### WS4: Exception Handler + Audit Log Table

**A. Exception handler** — `core/middleware.py`
- `app_exception_handler`: Log WARNING trước khi trả JSON (include request_id)
- Thêm catch-all `unhandled_exception_handler`: Log ERROR với `exc_info=True`, trả 500
- Include `request_id` trong error response JSON

**B. AuditLog table** — `infrastructure/database/models.py`

```
audit_logs:
  id          INTEGER PK AUTOINCREMENT
  timestamp   DATETIME (UTC)
  actor_email VARCHAR(255)    -- admin nào
  action      VARCHAR(50)     -- APPROVE, REJECT, DISABLE, ENABLE, DELETE
  entity_type VARCHAR(50)     -- service_type, service_instance, user
  entity_id   VARCHAR(150)    -- ID entity bị tác động
  details     TEXT (JSON)     -- action-specific data (vd: allowed_methods khi approve)
  request_id  VARCHAR(36)     -- correlation ID
  Indexes: timestamp, actor_email, (entity_type + entity_id), action
```

**C. AuditLogRepository** — File mới: `infrastructure/database/repositories/audit_log.py`
- `record(actor_email, action, entity_type, entity_id, details, request_id)`
- `query_by_entity(entity_type, entity_id, limit)`
- `query_recent(limit)`

**D. Record audit entries** — `api/v1/endpoints/admin/service_types.py`
- Gọi `audit_repo.record()` sau mỗi action (approve, reject, disable, enable, delete)

**E. Query endpoint** — `api/v1/endpoints/admin/dashboard.py`
- `GET /admin/dashboard/audit-log?entity_type=&entity_id=&limit=`
