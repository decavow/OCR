## Fix: Unicode filename gây 500 khi worker download file tiếng Việt

### Triệu chứng

```
ERROR: 'latin-1' codec can't encode character '\u1ec7' in position 25
POST /api/v1/internal/file-proxy/download HTTP/1.1 500 Internal Server Error
```

File có tên tiếng Việt (chứa ký tự `ệ`, `ấ`, `ở`...) → Starlette encode HTTP header bằng `latin-1` → crash.

### Root Cause

`file_proxy.py:47-54` đặt filename trực tiếp vào HTTP headers:
```python
headers={
    "Content-Disposition": f'attachment; filename="{filename}"',
    "X-File-Name": filename,
}
```
HTTP headers chỉ hỗ trợ ASCII/latin-1. Filename tiếng Việt chứa Unicode → `UnicodeEncodeError`.

### Fix

**Backend — `api/v1/internal/file_proxy.py`:**
- Dùng `urllib.parse.quote()` encode filename trước khi đặt vào header
- `Content-Disposition` dùng RFC 5987 format: `filename*=UTF-8''<encoded>`

**Worker — `clients/file_proxy_client.py`:**
- Dùng `urllib.parse.unquote()` decode filename khi nhận từ header

### Files thay đổi

| File | Loại | Thay đổi |
|------|:----:|----------|
| `02_backend/app/api/v1/internal/file_proxy.py` | Bugfix | URL-encode filename trong HTTP headers (RFC 5987) |
| `03_worker/app/clients/file_proxy_client.py` | Bugfix | URL-decode filename khi nhận từ header |

## Fix: Job state machine không match flow thực tế → SUBMITTED→FAILED bị reject

### Triệu chứng

```
WARNING: Invalid transition for job xxx: SUBMITTED -> FAILED
PATCH /api/v1/internal/jobs/xxx/status HTTP/1.1 404 Not Found
```

Worker download file bị 500 (Bug Unicode ở trên) → báo FAILED → backend reject vì `SUBMITTED → FAILED` không hợp lệ trong state machine.

### Root Cause

Flow thực tế: `SUBMITTED → (publish NATS) → Worker nhận → process → COMPLETED/FAILED`

Nhưng:
1. Upload service **không chuyển job sang QUEUED** sau khi publish thành công
2. Worker **không báo PROCESSING** trước khi bắt đầu xử lý
3. Job vẫn ở `SUBMITTED` khi worker báo kết quả → state machine reject

State machine: `SUBMITTED → {VALIDATING, REJECTED}` — thiếu path đến `QUEUED`.

### Fix

**3 chỗ sửa để flow khớp state machine:**

1. **State machine** (`modules/job/state_machine.py`):
   - Thêm `QUEUED` vào allowed transitions từ `SUBMITTED`
   - `SUBMITTED → {VALIDATING, QUEUED, REJECTED}`

2. **Upload service** (`modules/upload/service.py`):
   - Sau publish NATS thành công → chuyển job sang `QUEUED`

3. **Worker** (`03_worker/app/core/worker.py`):
   - Trước khi bắt đầu process → báo `PROCESSING` cho backend

**Flow sau fix:** `SUBMITTED → QUEUED → PROCESSING → COMPLETED/FAILED`

### Files thay đổi

| File | Loại | Thay đổi |
|------|:----:|----------|
| `02_backend/app/modules/job/state_machine.py` | Bugfix | Thêm `QUEUED` vào transitions từ `SUBMITTED` |
| `02_backend/app/modules/upload/service.py` | Bugfix | Chuyển job sang `QUEUED` sau publish NATS |
| `03_worker/app/core/worker.py` | Bugfix | Báo `PROCESSING` trước khi bắt đầu xử lý job |

## Fix: AccessDenied exception không nhận message → TypeError crash file download

### Triệu chứng

```
TypeError: AccessDenied.__init__() takes 1 positional argument but 2 were given
POST /api/v1/internal/file-proxy/download HTTP/1.1 500 Internal Server Error
```

Mọi request download file từ worker đều trả 500.

### Root Cause

`file_proxy/exceptions.py`: `AccessDenied.__init__(self)` và `ServiceNotRegistered.__init__(self)` không nhận argument.
Nhưng `access_control.py` gọi `raise AccessDenied("Job not found or not active")` truyền message → `TypeError`.

### Fix

Cho `AccessDenied` và `ServiceNotRegistered` nhận optional `message` parameter với default value.

### Files thay đổi

| File | Loại | Thay đổi |
|------|:----:|----------|
| `02_backend/app/modules/file_proxy/exceptions.py` | Bugfix | `AccessDenied` và `ServiceNotRegistered` nhận optional message |

## Fix: Race condition SUBMITTED→PROCESSING + PaddleOCR table matcher crash

### Bug 1: Race condition — Worker nhận job trước khi backend chuyển QUEUED

Upload service publish NATS **trước** khi update status → QUEUED.
Worker nhận message và gọi SUBMITTED→PROCESSING → bị reject.

**Fix**: Chuyển `update_status(QUEUED)` lên **trước** `queue.publish()` trong `upload/service.py`.

### Bug 2: PaddleOCR PPStructure table matcher crash trên ảnh không có table

```
IndexError: too many indices for array: array is 1-dimensional, but 2 were indexed
```

PPStructure `matcher.py` crash khi `pred_bboxes` rỗng/1D. Bug nội bộ PaddleOCR.

**Fix**: Thêm fallback engine (`table=False`) trong `handler.py`. Khi engine chính crash IndexError/ValueError → tự retry bằng fallback.

### Bug 3: Postprocessing crash khi fallback engine trả `res` không phải dict

```
AttributeError: 'str' object has no attribute 'get'
```

Khi `table=False`, PPStructure trả `res` là list/string thay vì dict. Postprocessing cần handle cả 3 format.

**Fix**: `extract_regions()` xử lý `res` theo type (dict/list/str).

### Files thay đổi

| File | Loại | Thay đổi |
|------|:----:|----------|
| `02_backend/app/modules/upload/service.py` | Bugfix | QUEUED trước publish NATS (fix race condition) |
| `03_worker/app/engines/paddle_vl/handler.py` | Bugfix | Fallback engine table=False khi table matcher crash |
| `03_worker/app/engines/paddle_vl/postprocessing.py` | Bugfix | Handle res dạng dict/list/str trong extract_regions |

## Fix: Fallback OCR cho kết quả quá kém (585 bytes cho PDF 8 trang)

### Triệu chứng

PDF `1709.04109v4.pdf` (8 pages) xử lý qua fallback chỉ trả 585 bytes — gần như rỗng.
So sánh: cùng file, fallback cũ (PPStructure table=False) cho 28,245 bytes.

### Root Cause

Commit trước đổi fallback từ `PPStructure(table=False)` sang `PaddleOCR` (plain OCR).
Plain PaddleOCR chỉ detect + recognize text, **mất toàn bộ layout analysis** (text/title/table/figure regions).
→ Postprocessing nhận format khác → hầu hết content bị bỏ qua.

### Fix

`handler.py`: Đổi fallback về `PPStructure(table=False)` — giữ layout analysis, chỉ tắt table recognition (nơi bug xảy ra).

- Xoá `PaddleOCR` import (không cần nữa)
- `_fallback_ocr` → `_fallback_engine` (PPStructure table=False, lazy-init)
- Xoá `_run_plain_ocr()` — không cần convert format thủ công
- `_run_engine()`: catch IndexError/ValueError → retry bằng fallback engine

### Files thay đổi

| File | Loại | Thay đổi |
|------|:----:|----------|
| `03_worker/app/engines/paddle_vl/handler.py` | Bugfix | Fallback dùng PPStructure(table=False) thay vì PaddleOCR plain |
